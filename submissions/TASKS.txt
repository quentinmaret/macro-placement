MACRO PLACEMENT PROJECT TASK PLAN

This document outlines the execution plan for building a modular macro placement system for the Partcl/HRT challenge. The focus is on clear task ownership, parallelization, and iterative improvement.

--------------------------------------------------
PROJECT GOAL
--------------------------------------------------

Build a modular pipeline:

initializer → legalization → hard-macro optimization → soft-macro refinement → evaluation

Each stage must be independently replaceable and testable.

--------------------------------------------------
PHASE 0 — CORE INFRASTRUCTURE
--------------------------------------------------

Objective:
Create a shared system that all other modules plug into.

Tasks:
- Wrap evaluation utilities into a unified API:
  - evaluate(placement)
  - validate(placement)
  - visualize(placement)
- Define a standard placement format (tensor / array structure)
- Build experiment runner:
  - run a placer across all benchmarks
  - log results
  - save placements and visualizations
- Add reproducibility:
  - seed control
  - config system

Deliverables:
- core/eval.py
- core/runner.py
- core/types.py

--------------------------------------------------
PHASE 1 — INITIALIZERS
--------------------------------------------------

Objective:
Generate diverse starting placements.

Tasks:
- Integrate provided random and greedy initializers
- Implement multi-start wrapper
- Add variations:
  - randomized ordering
  - slight noise perturbations

Deliverables:
- init/random.py
- init/greedy.py
- init/multistart.py

--------------------------------------------------
PHASE 2 — LEGALIZATION
--------------------------------------------------

Objective:
Ensure all placements are valid and repairable.

Tasks:
- Implement overlap detection
- Implement boundary enforcement
- Design repair strategy:
  - minimum displacement
  - prioritize large macros first
- Decide policy:
  - per-move legalization vs batch

Deliverables:
- core/legalize.py

--------------------------------------------------
PHASE 3 — CONNECTIVITY REPRESENTATION
--------------------------------------------------

Objective:
Capture macro relationships for better optimization.

Tasks:
- Build macro connectivity graph from nets
- Compute weighted edges
- Provide utilities:
  - neighbor lookup
  - macro importance metrics

Deliverables:
- core/graph.py

--------------------------------------------------
PHASE 4 — MOVE LIBRARY
--------------------------------------------------

Objective:
Define reusable local modifications.

Tasks:
- Implement:
  - single macro shift
  - macro swap
  - neighbor-directed move
  - random relocation
- Ensure moves are independent and fast

Deliverables:
- moves/basic.py
- moves/advanced.py

--------------------------------------------------
PHASE 5 — SEARCH ENGINE
--------------------------------------------------

Objective:
Optimize placements using moves.

Tasks:
- Implement simulated annealing or hill climbing
- Core loop:
  - propose move
  - legalize
  - evaluate
  - accept/reject
  - track best solution
- Add restart logic

Deliverables:
- optim/sa.py

--------------------------------------------------
PHASE 6 — SOFT MACRO REFINEMENT
--------------------------------------------------

Objective:
Optimize soft macros separately.

Tasks:
- Implement force-directed or equivalent refinement
- Experiment with schedules:
  - end-only
  - periodic
  - conditional

Deliverables:
- refine/soft_macro.py

--------------------------------------------------
PHASE 7 — EXPERIMENTATION FRAMEWORK
--------------------------------------------------

Objective:
Enable systematic comparison of approaches.

Tasks:
- Build ablation runner
- Compare:
  - initializers
  - move sets
  - refinement strategies
- Log:
  - scores
  - runtime
  - acceptance rates

Deliverables:
- experiments/ablation.py

--------------------------------------------------
PHASE 8 — PIPELINE COMPOSITION
--------------------------------------------------

Objective:
Combine modules into full systems.

Tasks:
- Build pipelines:
  - greedy → SA
  - random → SA → best
  - greedy → SA → refine
- Support easy swapping of modules

Deliverables:
- pipelines/*.py

--------------------------------------------------
PHASE 9 — LEARNING MODULES (OPTIONAL)
--------------------------------------------------

Objective:
Improve search efficiency using ML.

Tasks:
- Train models to:
  - rank moves
  - select macros
  - predict improvements

Deliverables:
- models/ranker.py

--------------------------------------------------
PHASE 10 — RL (OPTIONAL)
--------------------------------------------------

Objective:
Use RL as a high-level controller.

Tasks:
- RL selects:
  - macro
  - region
  - move type
- Keep search + refinement as execution layer

--------------------------------------------------
TEAM STRUCTURE
--------------------------------------------------

Team A — Infrastructure
Team B — Legalization
Team C — Optimization
Team D — Representation
Team E — Refinement
Team F — ML

--------------------------------------------------
EXECUTION PRIORITY
--------------------------------------------------

1. Infrastructure
2. Legalization
3. Move Library
4. Search Engine
5. Graph Representation
6. Soft Macro Refinement
7. Experiments
8. ML / RL

--------------------------------------------------
GUIDING PRINCIPLE
--------------------------------------------------

Build interfaces first, then implementations.

Every module must be swappable.
